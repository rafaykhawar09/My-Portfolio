What is the project?

The application allows the user to create, edit and delete reminders. Each hour of a day can have multiple reminders that are stored in the local storage. All the data stored in local storage for 24 hours, after which it is erased. Furthermore, each reminder can also be "checked off" if the users wishes to do so.


What I learnt & the challenge(s) I came across.


Everytime a reminder is entered into a time-block, a "new reminder" placeholder is appended to the time-block so that the user can enter another reminder if they wish to do so. As the div is appended on runtime we need event delegation for events such as "onSubmit" & "onClick" to work properly. I had previously worked with event delegation but not as extensively, so not only was this project a good reinforcement of my concepts, I also learnt other things, which leads me to my next challenge, traversing the DOM tree.


Syntactically traversing a DOM tree in vanilla javascript is different than traversing it in jQuery, which proved to be a slight hurdle. However, I enough practice of it as that is the main way I was accessing all the different elements.


I also learnt about javascript class objects, which work differently than classes from other languages that I'm used to mainly C++ and Java. It was primarily the syntax that took some getting use to.
