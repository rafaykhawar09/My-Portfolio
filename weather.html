What is the project?

It is a weather app concept for mobile phones. It shows the current weather, hourly and weekly forecast of the weather at a particular location. Openweather API is being used to get the weather data. I am also using illustrations from "unDraw" that I'm using to show the current weather condition.


The Process

I primarily used the iPhone's native weather app design as a starting point for the design of my own app. That said I added my own twists to it, being inspired by the numerous design concepts of weather apps on dribbble, and it is because of all that I saw there that made me want to include illustrations in my own app.

Challenge(s) I came across

Firstly, figuring out the design & layout for my application was challenging - deciding how and where the data would be shown was quite a tedious process.


Secondly, the Openweather API itself proved to be quite frustrating. At the time when I started coding the project, Openweather API did not provided a single API call to obtain all the data, i.e. the current weather and the hourly and weekly forecast. The user had to make three calls to obtain all the data, which was quite frustrating. However, they have recently provided an API call that can fetch all the required data, although I had to modify my code to accomodate for the change.


Lastly, as I'm not an illustrator I did not use my own illustrations, and as pleased as I am with the ones I found, there were times where I wished I had my own.


Future Improvements & Features

The very first feature I would add is the ability to search weather forecast of any city. At the moment, the application shows weather of where the user is currently located. I would probably be integrating Google's Map API for that and setting up a proxy server using Express.


I would also like to maintain the "history" of all the places the user has searched for and provide an easy access to to those locations, so that the user would not have to search for the same city over and over again.
